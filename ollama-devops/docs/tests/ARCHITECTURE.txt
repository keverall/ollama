Test Execution Graph

  lint ──→ shellcheck ──→ (PASS/FAIL)
    │
    ├─→ bash -n ──→ syntax valid?
    │
    ├─→ security scan ──→ eval/sudo/chmod777?
    │
    └─→ line endings ──→ LF?

  unit ──→ isolated function tests (mocked binaries)
    │
    └─→ fixture files / mock responses

  integration ──→ full script runs (mocked ollama/docker/nvidia)
    │
    └─→ validates workflow, log output, env vars

  smoke ──→ fast sanity checks on real binaries
    │
    └─→ verifies basic health

  e2e ───────────→ real hardware, full downloads
                  │
                  └─→ validates end-to-end correctness

Mock Layer Architecture

  tests/mocks/
  ├── ollama       ← substitutes real ollama binary
  ├── docker       ← substitutes docker CLI
  ├── docker-compose ← substitutes docker-compose
  ├── nvidia-smi   ← substitutes GPU detection
  └── curl         ← substitutes HTTP client

  Usage:
    export PATH="$PWD/tests/mocks:$PATH"
    export TEST_MOCK_ollama=always-fail   # control behavior
    ./scripts/sod.sh   ← sees mocks, not real binaries

  Benefits:
    ✓ No GPU required for unit tests
    ✓ No Docker daemon needed
    ✓ No 40GB model downloads
    ✓ Deterministic & fast (<5 sec)
    ✓ Can simulate errors easily
